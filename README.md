# 얼마낼까 (EolmaNaelkka) MVP

## 1. 프로젝트 개요
- **서비스명**: 얼마낼까
- **MVP 목표**: 한국소비자원(KCA)의 공신력 있는 데이터를 기반으로 '축의금 결정'이라는 단일 목적을 가장 빠르고 정확하게 해결한다.
- **핵심 가치**: "내 감이 아니라, 국가 통계가 알려주는 적정가"

## 2. 핵심 기능 (Minimal)
불필요한 기능(회원가입, 커뮤니티, 선물 추천 등)을 모두 배제하고 계산기 기능에만 집중합니다.

### ① 단일 페이지 계산기 (Single-Step Calculator)
- **지역 선택 (14개)**: 서울(강남), 서울(그 외), 경기/인천, 광역시, 그 외 지역 등 소비자원 분류 기준.
- **식사 여부**: 식사함 / 안 함(답례품) / 불참(봉투만).
- **친밀도**: 일반 지인(동료) / 친한 친구 / 아주 가까운 지인.
- **식장 유형**: 일반 웨딩홀 / 호텔(프리미엄).

### ② 데이터 결과 대시보드
- **권장 금액**: "추천 축의금: 100,000원" (가장 크게 노출).
- **산출 근거**:
  - "선택하신 지역의 평균 식대 중간값은 90,000원(강남 기준)입니다."
  - "식대 대비 약 10,000원의 축하금이 포함된 금액입니다."

## 3. 데이터 및 업데이트 설계
- **데이터 원천**: 한국소비자원 '12월 결혼서비스 가격 동향' (중간가격 데이터 활용).
- **데이터 관리**: `stats.json` 파일 하나로 관리 (서버/DB 비용 0원).
- **업데이트 프로세스**:
  1. **반자동 체크**: 매월 1회 소비자원 '참가격' 사이트의 보도자료 업로드 확인.
  2. **데이터 갱신**: 새 PDF 파일의 수치만 JSON 파일에 덮어쓰기.
  3. **배포**: GitHub 저장소에 Push하면 Vercel을 통해 10초 내로 자동 반영.

## 4. 디자인 가이드 (Minimalist UI)
모바일에서 '봉투 적기 전 30초' 만에 끝내는 UX를 지향합니다.

- **레이아웃**: 100% 한 페이지(Single Page Application). 스크롤 없이 모든 버튼이 한 화면에 보임.
- **컴포넌트**: Selection - 텍스트 입력 없이 **큰 버튼(Card)**을 탭하는 방식.
- **Typography**: 권장 금액은 **48px 이상의 볼드체**로 강조.
- **Color**:
  - **Background**: White (`#FFFFFF`)
  - **Primary**: Deep Navy (`#1A2B4B`) - 신뢰감 부여
  - **Point**: Green (`#28A745`) - '적정함'을 의미하는 신호등 색상
- **배너**: 하단에 작은 뱃지로 "대한민국 소비자원 2026.01.30 공식 데이터 기반" 명시.

## 5. 기술 스택 (Zero Cost)
- **Framework**: Next.js (Static Site Generation 활용으로 로딩 속도 극대화).
- **Deployment**: Vercel (Hobby 플랜 - 무료).
- **Domain**: 가비아 또는 Namecheap에서 '.com' 또는 '.kr' 구매 (연 2만 원 내외).

## 6. 검증 지표
- **이탈률**: 접속 후 결과 확인까지 20초 이내에 완료되는가?
- **공유 수**: 결과 하단의 '카카오톡 공유' 버튼 클릭 횟수. (데이터 기반 결과에 사람들이 얼마나 동감하는지 척도)
